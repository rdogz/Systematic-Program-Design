# Systematic Program Design

Before writing a single line of code, I learned to *think*. This repository follows my journey through **Systematic Program Design**. This course teaches you to design programs the right way: data first, structure second, code last.

## What This Course Covers

Three design recipes applied consistently across every problem:

- **HtDF** — How to Design Functions: signature, purpose, stub, tests, then code.
- **HtDD** — How to Design Data: define your data before your logic.
- **HtDW** — How to Design Worlds: event-driven interactive programs with `big-bang`.

Topics progress from basic expressions all the way to recursive algorithms, abstract functions, backtracking search, and cyclic graph traversal.
